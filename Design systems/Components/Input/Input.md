# Input

Часть [Design systems](../..). Инвентарь собран из Figma-файла [Components](https://www.figma.com/design/j5KLNIldIndVDcg5bsHZhC/Components), страница **Input**.

> Компонент для ввода текстовой информации. Представляет собой однострочное текстовое поле, которое позволяет пользователю вводить и редактировать текст.

## Размеры (px)

| Элемент | Значение |
|---|---|
| Поле, Size=Small (только сам инпут, без Label/Description) | высота 32px |
| Поле, Size=Medium (только сам инпут, без Label/Description) | высота 40px |
| Кликабельная область кнопки "очистить" (крестик) | 24×24px |
| Область ховера иконки в инпуте | 24×24px |
| Максимальная длина Description | ~140 символов |

Высота 32px/40px — это подтверждённое в самой Figma правило: "если кнопка стоит рядом с инпутом, кнопка должна быть размера как инпут, чтобы высота была одинакова" (Small-инпут ставится рядом с 32px-кнопкой, Medium — с 40px-кнопкой). Инстансы Input с Label/Description/Error поверх поля дают бóльшую суммарную высоту (64–122px в разных примерах) — это уже сумма поля + доп. элементов, а не отдельный токен. Ширина везде — по контейнеру (auto), фиксированного значения нет.

## Семейства

### Input
- **Type**: Gray / Outlined
- **Size**: Small / Medium
- **State**: Default / Hover / Active / Filled / Filled Hover / Filled Active / Disabled / Error

Внутри инпута можно независимо включать/выключать: Description, Label, Link (если есть Label), Icon left, Icon right. После ввода символа появляется кнопка-крестик для очистки поля. Если текст лейбла/инпута не помещается в строку — уходит в многоточие (в файле явно отмечено, что так делать "очень плохо", то есть это нежелательный кейс, а не задуманное поведение).
