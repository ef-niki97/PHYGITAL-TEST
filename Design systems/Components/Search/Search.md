# Search

Часть [Design systems](../..). Инвентарь собран из Figma-файла [Components](https://www.figma.com/design/j5KLNIldIndVDcg5bsHZhC/Components), страница **Search**.

## Размеры (px)

| Вариант | Высота | Ширина |
|---|---|---|
| Search (атомарный компонент) | 44px | 392px (пример) |
| Search — демо-примеры State=Default/Hover/Focus-visible × Type=Placeholder/Filled | 61px (пример с подписью) | 66–169px (по контенту) |

Есть один "чистый" символ Search (44px высотой, без подписи) — его стоит считать основным токеном высоты. Второй набор (61px) — это композиция с текстовой подписью-лейблом над полем в демо-раскладке, а не размер самого поля.

## Семейства

### Search
- **State**: Default / Hover / Focus-visible
- **Type**: Placeholder / Filled

Назначение не задокументировано в Figma (нет блока "Для чего применяется?").

## Реализация в коде (продакшен-референс)

Путь в исходном репозитории: `src/solidJs/shared/ui/molecules/SearchableMenu`

Уточнение: точного соответствия "поле поиска" в коде нет — `SearchableMenu` это асинхронный комбобокс с автокомплитом (поиск + выбор из списка), построенный на `Select` из `@ark-ui/solid`. Поле ввода находится не в триггере, а внутри выпадающего списка (sticky сверху над результатами) и управляет запросом через `@tanstack/solid-query` (`useQuery`) с debounce по `staleTime`. Реализованы состояния: загрузка (`Spinner`), ошибка (`ErrorMessage`), пустой результат, подсказка "введите ещё символы" при длине строки меньше `minSearchLength` (по умолчанию 2), и обычный список найденных пунктов с отметкой выбранного (`Check`). Есть два экспорта — `SearchableMenu` (одиночный выбор, с кнопкой очистки) и `SearchableMenuMultipple` (множественный выбор, выбранные пункты рендерятся отдельными чипами с крестиком для снятия). Пропсы: `name`, `labelContent`, `value`, `onChange`, `errorMessage`, `searchFn` (асинхронная функция поиска), `extract` (функции извлечения id/label/name из объекта), `placeholders` (тексты для триггера/поиска/пустого состояния/ошибки), `minSearchLength`. Расхождение с Figma-инвентарём: в Figma заявлены состояния Default/Hover/Focus-visible и Type=Placeholder/Filled для самого поля — в коде эти визуальные состояния относятся к `Input` внутри выпадающего меню и к `Trigger` кнопке, а не к отдельному полю поиска как таковому.

Скопированные файлы:
- `Reference Code/index.tsx` — компоненты `SearchableMenu` и `SearchableMenuMultipple`: async-поиск, автокомплит, состояния загрузки/ошибки/пустого результата, одиночный и множественный выбор.
