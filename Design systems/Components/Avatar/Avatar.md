# Avatar

Часть [Design systems](../..). Инвентарь собран из Figma-файла [Components](https://www.figma.com/design/j5KLNIldIndVDcg5bsHZhC/Components), страница **Avatar**.

> Используется для визуального представления пользователя, сущности или источника контента.

## Размеры (px)

Все размеры — квадратные (высота = ширина), фиксированы для каждого шага шкалы:

| Size | Высота/Ширина |
|---|---|
| XS | 24px |
| S | 32px |
| M | 40px |
| L | 52px |
| XL | 60px |
| 2XL | 75px |
| 3XL | 100px |

## Семейства

### Avatar
- **Size**: 24 / 32 / 40 / 52 / 60 / 75 / 100
- **Type**: Image / Text / Icon
- **Circle**: On / Off

Полная матрица Size × Type × Circle задокументирована для всех 7 размеров.

## Реализация в коде (продакшен-референс)

Путь в исходном репозитории: `src/solidJs/shared/ui/atoms/Avatar`

В коде базовый `Avatar` — это `Button` (см. компонент Buttons) со стилями из `tv`: `size-10 rounded-[10px] p-2.5` для `md` и `rounded-2xl p-3` (без фиксированной ширины/высоты) для `lg`. Реализовано только 2 значения size (`md`/`lg`) против 7 задокументированных в Figma (24/32/40/52/60/75/100) — заметное расхождение, размерная шкала в коде явно уже, чем в дизайне. Проп `Circle` как переключатель отсутствует: форма (скругление) жёстко привязана к размеру, а не переключается отдельным флагом. Из типов реализованы `GptAvatar` (иконка в круглой рамке) и `UserAvatar` (первая буква имени пользователя через `getAvatarLetter`, т.е. вариант Text) — вариант Image (аватар из изображения) в скопированном файле не реализован.

Скопированные файлы:
- `Reference Code/Avatar.tsx` — базовый `Avatar` (обёртка над `Button`), `GptAvatar` (иконка), `UserAvatar` (буква имени), tailwind-variants для size md/lg
