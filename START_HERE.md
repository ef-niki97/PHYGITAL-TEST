# START HERE — гайд для агента

Это репозиторий дизайн-документации **Phygital+** (AI-редактор на SolidJS с node-based холстом для генерации изображений/видео). Здесь нет кода — только правила, гайдлайны и контекст, по которым нужно сверяться при любой дизайн- или продуктовой задаче. Ниже — 10 пунктов, на которые нужно обратить внимание в первую очередь.

1. **Дизайн-система — источник истины, а Figma — только для исследования.** Сначала ищи существующий компонент/паттерн (`Design systems/Components/`), а не изобретай новый. Подробности приоритетов решений — [Rules/Design Principles.md](Rules/Design%20Principles.md).

2. **Только токены, никаких произвольных значений.** Отступы — исключительно из шкалы 4/8/12/16/24/32/40/48/64px. Цвета — только семантические (Text/Primary, Background/Gray-light и т.п.), не хардкод-hex. Иконки — только Lucide, рекомендуемые размеры 16/20/24/32/44/48px. См. [Rules/Design Principles.md](Rules/Design%20Principles.md).

3. **Простота важнее эффектов.** Каждый элемент должен решать конкретную задачу и снижать когнитивную нагрузку; одноколоночная структура, минимум вложенности, до 3 размеров шрифта на компонент. Перед изменением экрана — задай себе контрольные вопросы из раздела «Перед любым изменением спроси себя» в том же файле.

4. **Продукт — node-based, не линейный.** Интерфейс строится вокруг бесконечного холста и нод (карточка ноды: radius 24px, генерация — контрастная чёрная кнопка на всю ширину, порты снаружи слева/справа, точечная сетка 16px, snap 8px). Не предлагай линейные wizard-паттерны там, где логика — canvas. См. [Rules/UI_UX_Guidelines.md](Rules/UI_UX_Guidelines.md).

5. **Accessibility обязателен, не опционален.** Контраст WCAG AA (4.5:1), минимальная интерактивная область 44×44px, полная доступность с клавиатуры, состояние никогда не передаётся только цветом (порты нод — цвет + тултип с подписью).

6. **Продуктовая философия — workflow-first, а не feature/model-first.** Решения встраиваются в реальный рабочий процесс пользователя, а не добавляются поверх него; UI строится вокруг типов контента (модальностей), а не конкретных AI-моделей. Полное раскрытие — [Playbook/Product-Vision.md](Playbook/Product-Vision.md).

7. **Тексты и тон: Phygital+ усиливает дизайнера, а не заменяет.** Запрещённый угол подачи — «AI designer replacement». Балансируй anxiety (низкий порог входа) и excitement (контроль результата). Пиши коротко, без маркетингового пафоса. См. [Rules/Copywriting.md](Rules/Copywriting.md).

8. **Задачи любой сложности проходят по 10-шаговому процессу** (постановка → референсы → концепты → варианты → финальное решение → сборка на компонентах → ревью → задача на разработку → проверка → финализация), а не решаются в одно действие в обход процесса. См. [Playbook/Design-Process-Flow.md](Playbook/Design-Process-Flow.md).

9. **У интерактивных элементов обязательны все состояния.** Hover, focus, active, disabled — не только «дефолт», иначе элемент нельзя считать готовым. См. [Playbook/Playbook.md](Playbook/Playbook.md).

10. **Не переделывай экран целиком без необходимости.** Сохраняй существующие UX-паттерны продукта, выбирай решение с минимальными изменениями, если вариантов несколько. Для работы с реальными Figma-файлами — плагин Figma уже установлен и активен, скиллы триггерятся автоматически (см. [Skills/Figma.md](Skills/Figma.md)).

## Структура папки

- `Rules/` — обязательные к соблюдению правила (дизайн-принципы, UI/UX гайдлайны продукта, копирайтинг).
- `Playbook/` — процессы команды, продуктовое видение, манифест, ресурсы.
- `Design systems/Components/` — спецификации существующих UI-компонентов.
- `Skills/` — справки по установленным AI-скиллам (Figma и др.).
- `README.md` — техстек и архитектура кодовой базы (SolidJS, Feature-Sliced Design).
