# Принципы дизайна

- Простота важнее визуальных эффектов.
- Каждый элемент интерфейса должен решать конкретную задачу.
- Минимизируй когнитивную нагрузку пользователя.
- Показывай только ту информацию, которая нужна на текущем этапе.
- Предпочитай постепенное раскрытие сложных функций, а не перегруженные экраны.
- Последовательность важнее оригинальности.

# Компоновка

- Предпочитай одноколоночную структуру.
- Не создавай лишних уровней вложенности.
- Группируй связанные элементы.
- Используй отступы для разделения блоков вместо большого количества рамок.
- Главный сценарий должен сразу считываться взглядом.
- Основное действие должно быть самым заметным.

# Типографика

- Не используй более трёх размеров шрифта внутри одного компонента.
- Используй обычный регистр текста.
- Жирное начертание используй только для создания иерархии.
- Заголовки должны описывать содержимое раздела.
- Подписи должны быть короткими и понятными.

# Компоненты

- Всегда сначала используй существующие компоненты дизайн-системы.
- Не создавай новый компонент, если текущий можно адаптировать.
- Не добавляй новые варианты компонентов без необходимости.
- Предпочитай композицию существующих компонентов созданию новых.

# Отступы

Используй только значения из дизайн-системы.

Допустимые значения:

4
8
12
16
24
32
40
48
64

Не используй произвольные значения.

# Цвет

- Используй только цвета дизайн-системы.
- Не добавляй новые цвета.
- Основной цвет используй только для:
    - главных действий;
    - активных состояний;
    - выбранных элементов.
- Не используй цвет как единственный способ передачи информации.

# Иконки

- Используй только Lucide Icons.
- Не смешивай разные стили иконок.
- Используй рекомендуемые размеры — 16х16, 20х20, 24х24, 32х32, 44х44, 48х48px.

# UX-тексты

- Пиши коротко.
- Кнопки должны содержать действие.
- Избегай маркетинговых формулировок.
- Используй простой язык.
- Если текст можно сократить без потери смысла — сокращай.
- При написании текста смотри в документ copywrite.md

# Empty State

Каждое пустое состояние должно отвечать на три вопроса:

- Что произошло?
- Почему это произошло?
- Что пользователь может сделать дальше?

# Поведение AI

- Не переделывай экран полностью без необходимости.
- Сохраняй существующие UX-паттерны продукта.
- Предпочитай небольшие улучшения полной переработке.
- Если есть несколько решений — выбирай то, которое требует минимальных изменений.

# Перед любым изменением спроси себя

- Можно ли использовать существующий компонент?
- Можно ли убрать этот элемент?
- Можно ли объединить несколько действий в одно?
- Можно ли сделать экран проще?
- Можно ли уменьшить количество визуального шума?
- Несёт ли новый элемент реальную пользу пользователю?

# Accessibility

- Соблюдай достаточный контраст.
- Минимальный размер интерактивной области — 44×44 px.
- Интерфейс должен быть доступен с клавиатуры.
- Не полагайся только на цвет для передачи состояния.

# Приоритет решений

1. Использовать существующую дизайн-систему.
2. Следовать существующим паттернам продукта.
3. Следовать платформенным рекомендациям.
4. Только после этого использовать общие UX-практики.

Никогда не нарушай этот порядок.

# Характер продукта

Наш продукт:

- спокойный, а не яркий;
- функциональный, а не декоративный;
- минималистичный, но не пустой;
- плотный по информации, но легко сканируемый;
- профессиональный, а не развлекательный;
- рассчитан на ежедневную работу опытных пользователей.

# Подход к проектированию

- Упрощай, а не усложняй.
- Каждое новое действие должно быть оправдано.
- Каждый новый элемент должен приносить ценность.
- Сначала убирай лишнее, потом добавляй новое.
- Скорость работы опытного пользователя важнее эффектных решений.
- Интерфейс должен помогать выполнять задачу, а не привлекать внимание.
- Если решение вызывает сомнения — выбирай более простое.
