# Figma (плагин Claude Code)

Официальный плагин Figma для Claude Code: MCP-сервер + набор скиллов для работы с Figma-файлами. Уже установлен и активен (маркетплейс Anthropic, `claude-plugins-official/figma`, v2.2.81) — отдельная установка не требуется. Этот файл — справка о том, что умеет плагин и когда какой скилл вызывается автоматически.

## Направление: код → Figma

- **figma-generate-design** — собирает экраны, модалки, дровера, сайдбары и другие многосекционные view в Figma из кода/описания, переиспользуя опубликованную дизайн-систему (компоненты, переменные, стили) вместо примитивов с хардкодными значениями.
- **figma-generate-library** — строит или обновляет дизайн-систему в Figma по кодовой базе: токены/переменные, библиотека компонентов с вариантами, тема (light/dark), фундамент. Используется даже для одного компонента.
- **figma-create-new-file** — создаёт новый пустой файл Figma (Design, FigJam или Slides) перед началом работы.
- **figma-use** — базовый скилл для `use_figma`: выполнение JS через Plugin API (создание/правка нод, переменные, авто-layout, компоненты). Обязателен перед любым вызовом `use_figma`.
- **figma-use-figjam / figma-use-slides / figma-use-motion** — доп. контекст к `figma-use` для FigJam, Slides и анимаций соответственно.
- **figma-generate-diagram** — превращает Mermaid-синтаксис в редактируемую диаграмму в FigJam (флоучарты, архитектура, ERD, sequence, gantt и т.д.).
- **figma-implement-motion** — переносит анимации/переходы из Figma в рабочий код (CSS keyframes, motion.dev и т.п.).

## Направление: Figma → код

- **figma-design-to-code** — реализует выбранный Figma-дизайн как код в целевом проекте (`get_design_context`).
- **figma-code-connect** — создаёт/поддерживает Code Connect шаблоны (`.figma.ts`), которые связывают компоненты Figma с реальными сниппетами кода.
- **figma-swiftui** — двусторонний мост Figma ↔ SwiftUI (дизайн → код и код → дизайн) для iOS/iPadOS.

## Как это вызывается

Скиллы триггерятся автоматически по контексту запроса (упоминание Figma-ссылки, "собери экран в Figma", "перенеси анимацию в код" и т.п.) — вручную вызывать не нужно, Claude сам подгружает нужный скилл перед соответствующим MCP-инструментом (`use_figma`, `get_design_context`, `create_new_file`, `generate_diagram`).

## Требования

- Подключение к Figma MCP-серверу должно быть авторизовано (OAuth) — если недоступно, Claude сообщит об этом и попросит авторизовать через `/mcp` или настройки коннекторов.
- Лимиты запросов на чтение зависят от плана Figma (Starter/View/Collab — до 6 вызовов в месяц; Dev/Full seat на платных планах — лимиты как у Figma REST API). Запись в файлы (write to canvas) под лимиты не подпадает.
